üìÇ 1. Understanding Permissions (Symbolic & Octal)

Each file and directory has three sets of permissions:

  • User (u) ‚Äì File owner
  • Group (g) ‚Äì Users in the file's group
  • Others (o) ‚Äì All other users
ls -l
-rwxr-xr-- 1 alice staff 1234 Apr 1 14:00 script.sh

In this example:

  • rwx ‚Üí owner has read/write/execute
  • r-x ‚Üí group has read/execute
  • r-- ‚Üí others have read only

üî¢ 2. Octal and Symbolic Notation

Octal Permissions:

7rwxRead, Write, Execute

chmod 755 file       # rwxr-xr-x
chmod 644 file       # rw-r--r--
chmod u+x file       # Add execute to user
chmod g-w file       # Remove write from group
chmod o=r file       # Others: read only

üîí 4. Special Permissions

  • SUID: Executes with owner‚Äôs privileges
  • SGID: Inherits group for new files or executes as group
  • Sticky Bit: Restricts deletion in shared dirs like /tmp

chmod u+s file          # Set SUID
chmod g+s file          # Set SGID
chmod +t /shared/dir    # Set sticky bit

Results in:


-rwsr-xr-x file      # SUID (s in user exec)
drwxrwsr-x dir       # SGID (s in group exec)
drwxrwxrwt dir       # Sticky Bit (t in others exec)
